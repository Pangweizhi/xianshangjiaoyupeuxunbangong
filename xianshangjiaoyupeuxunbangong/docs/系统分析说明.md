# 系统分析说明

## 1. 文档说明

本文根据当前仓库中的后端代码、前端工程、数据库脚本和文档做静态梳理，说明系统主题、角色、功能、技术栈和当前实现现状。

截至 `2026-05-07`，当前仓库已经不是单纯的“课程 + 作业 + 公告”基础系统，而是扩展为包含课程学习、进度、考试、结课、学分、智能问答能力和题库 AI 出题能力的综合教学平台。

## 2. 系统主题

该系统是一个“线上教育培训办公系统”，目标是把学生、教师、管理员三类角色的教学、学习和运营工作放到同一套平台中完成。

当前覆盖的核心业务包括：

- 课程管理与课程审核
- 课程章节与课程资源管理
- 学生选课与我的课程
- 视频学习与学习进度跟踪
- 作业发布、提交、批改
- 考试、题库、阅卷、成绩
- 结课判定与学分发放
- 公告与公告类型管理
- 智能问答助手
- 题库 AI 出题

## 3. 角色体系

系统当前可确认的角色有 3 类：

- 管理员
- 教师
- 学生

### 3.1 管理员

管理员账号来自 `users` 表，负责平台级管理、审核、字典维护和全局业务数据管理。

### 3.2 教师

教师账号来自 `jiaoshi` 表，负责课程、章节、资源、作业、考试等教学业务维护。

### 3.3 学生

学生账号来自 `yonghu` 表，负责选课、学习、作业提交、参加考试、查看成绩等学习行为。

## 4. 鉴权与权限现状

当前系统仍沿用原有后端的基于 `token` 与 `session` 的鉴权方式：

- 登录后返回 `token`
- 后端通过 `AuthorizationInterceptor` 做身份恢复
- 很多列表接口会根据当前角色自动过滤数据范围

前端层面则通过角色控制菜单和入口：

- 管理员看到后台完整菜单
- 教师看到教学相关菜单
- 学生看到学习相关页面

真正的数据权限仍由后端控制。

## 6. 技术栈现状

### 6.1 后端

后端是典型的 Spring Boot 单体项目，当前可确认的技术栈包括：

- `Spring Boot 2.2.2.RELEASE`
- `MyBatis-Plus 2.3`
- `mybatis-spring-boot-starter 2.1.1`
- `Spring JDBC`
- `MySQL Connector`
- `Shiro 1.3.2`
- `Fastjson`
- `Hutool`
- `Apache POI`

### 6.2 新前端

新前端位于 `new-frontend`，采用 `pnpm workspace` 管理，拆分为：

- `front-web`
- `admin-web`
- `shared`

其中：

- `front-web` 使用 `Vue 3 + Vite + TypeScript + Pinia + Vue Router`
- `admin-web` 使用 `Vue 3 + Vite + TypeScript + Element Plus + Pinia + Vue Router`
- `shared` 负责共享类型、HTTP 封装与通用逻辑

### 6.3 旧前端

仓库中仍保留旧前端资源：

- 旧学生端静态资源
- 旧后台资源

当前策略不是直接删除旧前端，而是在 `new-frontend` 中持续建设新前端。
