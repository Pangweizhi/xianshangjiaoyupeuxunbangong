# 系统分析说明

本文根据当前仓库中的后端代码、前端资源、`pom.xml`、`application.yml` 和 `db.sql` 做静态梳理，整理系统主题、角色、功能、技术栈和数据库现状。

## 1. 系统主题

该系统是一个**线上教育培训办公系统**，核心目标是把学生、教师和管理员的日常业务放到同一套平台中管理。

从当前源码看，系统主要覆盖以下业务场景：

- 学生注册、登录、资料维护
- 教师注册、登录、资料维护
- 课程、备课信息、作业、会议通知、公告等教学管理
- 论坛交流
- 学生与教师之间的电访/沟通记录
- 基础字典、系统配置、文件上传、数据库备份等后台管理能力

## 2. 角色

当前系统里能确认的角色有 3 类：

- `管理员`
- `教师`
- `学生`

### 2.1 管理员

管理员账号在 `users` 表中，负责平台级管理。

### 2.2 教师

教师账号在 `jiaoshi` 表中，属于教学业务侧的核心角色。

### 2.3 学生

学生账号在 `yonghu` 表中，属于平台的学习与参与侧角色。

## 3. 角色之间的关系

系统采用 `token` + `session` 的方式识别角色。

- `token` 表保存 `userid`、`username`、`tablename`、`role`
- 登录成功后，后端会把 `userId`、`role`、`tableName`、`username` 写入 session
- `AuthorizationInterceptor` 会拦截请求，并根据 token 恢复当前用户身份

角色关系不是简单并列，而是有明显的数据归属关系：

- 管理员可以查看和管理全局数据
- 教师可以管理与自己相关的教学业务数据
- 学生主要查看和维护自己的数据，以及参与论坛、查看课程和公告等

从控制器代码可以看到，很多列表接口会按当前 session 里的 `role` 自动过滤：

- 学生只看自己的学生相关数据
- 教师只看自己的教师相关数据
- 管理员则通常可看全量数据

论坛数据是一个典型的三角色关联模型：

- `forum.yonghu_id` 对应学生
- `forum.jiaoshi_id` 对应教师
- `forum.users_id` 对应管理员

- 发布作业
- 发布会议通知

## 6. 技术栈现状

### 6.1 后端

后端是典型的 Spring Boot 单体项目，当前 `pom.xml` 中能确认的技术栈如下：

- `Spring Boot 2.2.2.RELEASE`
- `MyBatis-Plus 2.3`
- `mybatis-spring-boot-starter 2.1.1`
- `Spring JDBC`
- `MySQL Connector`
- `Shiro 1.3.2` 依赖
- `Fastjson`
- `Hutool`
- `Commons Lang3`
- `Commons IO`
- `Apache POI`
- `Baidu AI SDK`

后端运行配置：

- 端口：`8080`
- 上下文路径：`/xianshangjiaoyupeuxunbangong`
- 数据库：`127.0.0.1:3306/xianshangjiaoyupeuxunbangong`

后端结构特点：

- `com.controller`：控制器
- `com.service` / `com.service.impl`：业务层
- `com.dao`：数据访问层
- `com.entity` / `com.entity.vo` / `com.entity.view`：实体与视图对象
- `com.interceptor.AuthorizationInterceptor`：Token 权限拦截

### 6.2 前端

当前仓库里有两套前端资源形态：

#### 后台管理端

后台管理端是一个完整的 Vue2 工程，位于：

- `src/main/resources/admin/admin`

从 `package.json` 可确认的前端技术栈：

- `Vue 2.6.10`
- `Vue Router 3.x`
- `Element UI 2.15.10`
- `Axios`
- `ECharts`
- `vue-quill-editor`
- `vue-amap`
- `vue-qr`
- `vue-json-excel`
- `print-js`
- `vue-cli 4.x`

页面组织上，后台管理端有：

- 登录、首页、个人中心、修改密码、支付页
- 各业务模块列表页和编辑页
- 字典管理页
- 数据库备份页

#### 前台展示端

前台展示端位于：

- `src/main/resources/front/front`

它更像是已经打包好的静态前台资源，而不是单独保留了完整源码工程。当前可见资源包括：

- `index.html`
- `pages/*/*.html`
- `css/*`
- `js/*`
- `layui/*`
- `xznstatic/*`

前台技术形态以以下内容为主：

- 传统静态 HTML/CSS/JS
- `Vue.js`
- `Element UI`
- `layui`
- `jQuery`
- `Bootstrap`
- `Swiper`
- `WOW.js`

### 6.3 前后台联动方式

前后台并不是两个独立后端，而是**同一个 Spring Boot 后端**统一提供资源和接口：

- 后端通过 `WebMvcConfigurationSupport` 映射静态资源目录
- `admin/admin` 和 `front/front` 都是通过后端打包进 `classpath` 的静态资源
- 前后台共用同一套接口和同一张数据库

## 7. 数据库现状

当前数据库脚本是 `db.sql`，数据库字符集为 `utf8`，表结构已经初始化并带有样例数据。

### 7.1 核心表

- `users`：管理员
- `yonghu`：学生
- `jiaoshi`：教师
- `token`：登录令牌
- `dictionary`：字典表
- `config`：系统配置
- `forum`：交流平台
- `kecheng`：课程
- `jiaoxueshipin`：备课信息
- `zuoye`：作业
- `kaihuitongzhi`：会议通知
- `news`：公告
- `yunyingshijian`：运营事件
- `jiaoshiqingjia`：教师请假
- `dianfnag`：电访信息

### 7.2 数据库设计特点

- 大量采用逻辑删除字段，例如 `*_delete`
- 多数业务表带有 `create_time`、`insert_time`、`update_time`
- 很多表通过 `*_types` 字段连接字典表
- 论坛、课程、作业、请假等模块都保留了与学生/教师/管理员的外键关系

### 7.3 当前数据现状

脚本中已经包含初始化数据，例如：

- 管理员账号
- 学生样例账号
- 教师样例账号

- 各业务表的样例记录

这意味着当前项目更偏向于“可直接运行演示”的教学管理系统模板。

## 8. 结论

综合现有源码可以确认，这是一套以**线上教育培训办公管理**为主题的系统，核心是围绕管理员、教师、学生三类角色展开。

系统特点可以概括为：

- 单体 Spring Boot 后端
- Vue2 管理端 + 静态前台页
- MyBatis-Plus 数据访问
- 角色权限主要靠 Token + Session 控制
- 业务上偏向教学资料、课程、公告、论坛、请假和作业管理
