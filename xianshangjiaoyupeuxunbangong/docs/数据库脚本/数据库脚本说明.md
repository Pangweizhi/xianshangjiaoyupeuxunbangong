# 数据库脚本说明

## 1. 当前脚本列表

当前仓库中和近期业务改造直接相关的脚本有两份：

- `docs/数据库脚本/zuoye_submit.sql`
- `docs/数据库脚本/核心业务一期.sql`

## 2. `zuoye_submit.sql` 的作用

该脚本用于补齐作业提交与批改能力，主要解决：

- 学生提交作业附件
- 教师批改
- 教师评分
- 教师评语
- 批改时间记录

核心字段包括：

- `submit_status`
- `submit_score`
- `submit_remark`
- `check_time`

## 3. `核心业务一期.sql` 的作用

该脚本用于支撑本轮新增的核心业务一期实现，主要内容包括：

- 扩展 `kecheng` 表
- 扩展 `zuoye` 表
- 新增 `course_chapter`
- 新增 `course_resource`
- 新增 `course_enroll`
- 新增 `study_progress`
- 新增 `course_credit_record`

这份脚本对应的业务能力是：

- 课程审核
- 章节管理
- 资源管理
- 学生选课
- 学习进度记录
- 作业关联课程/章节
- 学分发放记录

## 4. 建议执行顺序

建议按下面顺序执行：

1. 先执行基础库初始化脚本
2. 执行 `docs/数据库脚本/zuoye_submit.sql`
3. 执行 `docs/数据库脚本/核心业务一期.sql`

## 5. 当前代码对应关系

`zuoye_submit.sql` 主要对应：

- `ZuoyeSubmitController`
- `ZuoyeSubmitEntity`
- `ZuoyeSubmitDao.xml`

`核心业务一期.sql` 主要对应：

- `KechengController`
- `CourseChapterController`
- `CourseResourceController`
- `CourseEnrollController`
- `StudyProgressController`
- `CourseCreditRecordController`
- `CourseSettlementServiceImpl`
