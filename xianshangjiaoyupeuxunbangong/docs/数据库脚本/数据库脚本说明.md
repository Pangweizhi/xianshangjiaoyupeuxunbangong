# 数据库脚本说明

## 1. 当前脚本列表

与当前核心业务直接相关的脚本包括：

- `docs/数据库脚本/zuoye_submit.sql`
- `docs/数据库脚本/核心业务一期.sql`
- `docs/数据库脚本/核心业务三期.sql`
- `docs/数据库脚本/题库初始化.sql`
- `docs/数据库脚本/题库课程题目补充.sql`

## 2. `zuoye_submit.sql` 的作用

用于补齐作业提交与批改能力，主要覆盖：

- 学生提交作业附件
- 教师评分与评语
- 批改时间记录

主要字段包括：

- `submit_status`
- `submit_score`
- `submit_remark`
- `check_time`

## 3. `核心业务一期.sql` 的作用

用于支撑课程学习主链路，主要覆盖：

- 扩展 `kecheng`
- 扩展 `zuoye`
- 新增 `course_chapter`
- 新增 `course_resource`
- 新增 `course_enroll`
- 新增 `study_progress`
- 新增 `course_credit_record`

对应能力：

- 课程审核
- 章节管理
- 资源管理
- 学生选课
- 学习进度记录
- 作业关联课程/章节
- 学分发放记录

## 4. `核心业务三期.sql` 的作用

用于支撑考试与题库主链路，主要覆盖：

- 新增 `exam`
- 新增 `exam_question`
- 新增 `exam_record`
- 扩展考试规则字段

当前考试规则字段包括：

- `allow_retake`
- `max_attempt_count`
- `allow_resume`

## 5. `题库初始化.sql` 的作用

用于初始化题库基础样例数据。

如果库里还没有题库基础数据，建议先执行该脚本，再执行后续课程题目补充脚本。

## 6. `题库课程题目补充.sql` 的作用

该脚本是最近新增的题库补数脚本，用于按课程补充题目数据，并与当前代码中的“题库按课程分类”实现保持一致。

当前覆盖课程：

- 高等数学
- 新概念英语
- 大学语文
- 大学历史

每门课程补充：

- 选择题 `5` 道
- 判断题 `5` 道
- 填空题 `5` 道
- 简答题 `5` 道

合计：

- `4` 门课程
- `4` 种题型
- 每种 `5` 题
- 共 `80` 道题

## 7. `题库课程题目补充.sql` 的兼容处理

考虑到部分旧数据库结构不完整，该脚本已内置兼容逻辑：

- 先检查 `exam_question` 表中是否存在 `kecheng_id`
- 若不存在，则动态补充 `kecheng_id`
- 字段补齐后再执行插题

已规避的问题包括：

- `1054 Unknown column 'kecheng_id' in 'field list'`
- 旧版 MySQL 不支持 `ADD COLUMN IF NOT EXISTS`
- 顶层 `UNION ALL` 列数不一致导致的 `1222` 报错

## 8. 建议执行顺序

建议按照以下顺序执行：

1. 基础建库脚本
2. `docs/数据库脚本/zuoye_submit.sql`
3. `docs/数据库脚本/核心业务一期.sql`
4. `docs/数据库脚本/核心业务三期.sql`
5. `docs/数据库脚本/题库初始化.sql`
6. `docs/数据库脚本/题库课程题目补充.sql`

如果前面的结构已经存在，也可以只执行后面的补数脚本，但仍建议先确认：

- `exam_question` 表存在
- `kecheng` 表存在
- 课程基础数据中包含：
  - 高等数学
  - 新概念英语
  - 大学语文
  - 大学历史

## 10. 使用注意事项

- 如果课程名称与脚本中写死的课程名称不一致，按课程名称关联时可能插不进数据
- 如果已经执行过相同补数脚本，需要先确认是否会产生重复题目
- 建议优先在测试库执行，再导入正式库
