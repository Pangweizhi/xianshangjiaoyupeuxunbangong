# 数据库脚本说明

## 1. 当前脚本列表

当前与核心业务改造直接相关的脚本有三份：

- `docs/数据库脚本/zuoye_submit.sql`
- `docs/数据库脚本/核心业务一期.sql`
- `docs/数据库脚本/核心业务三期.sql`
- `docs/数据库脚本/题库初始化.sql`

## 2. `zuoye_submit.sql` 的作用

该脚本用于补齐作业提交与批改能力，主要包括：

- 学生提交作业附件
- 教师评分与评语
- 批改时间记录

核心字段：

- `submit_status`
- `submit_score`
- `submit_remark`
- `check_time`

## 3. `核心业务一期.sql` 的作用

该脚本支撑课程学习主链路，主要内容包括：

- 扩展 `kecheng`
- 扩展 `zuoye`
- 新增 `course_chapter`
- 新增 `course_resource`
- 新增 `course_enroll`
- 新增 `study_progress`
- 新增 `course_credit_record`

对应业务能力：

- 课程审核
- 章节管理
- 资源管理
- 学生选课
- 学习进度记录
- 作业关联课程/章节
- 学分发放记录

## 4. `核心业务三期.sql` 的作用

该脚本支撑考试模块与考试规则控制，主要内容包括：

- 新增 `exam`
- 新增 `exam_question`
- 新增 `exam_record`
- 为 `exam` 增加考试规则字段

当前考试规则字段包括：

- `allow_retake`
- `max_attempt_count`
- `allow_resume`

## 5. `题库初始化.sql` 的作用

该脚本用于批量初始化题库样例，当前覆盖：

- 高等数学
- 新概念英语
- 大学语文
- 大学历史

每门课程包含：

- 选择题
- 判断题
- 填空题
- 简答题

当前代码还依赖这些记录状态：

- `started`
- `pending_check`
- `checked`
- `reset`
- `voided`

其中：

- `reset` 表示教师重置未提交作答，学生可重新开始
- `voided` 表示教师作废考试记录，不再计入有效次数和成绩

## 5. 建议执行顺序

建议按以下顺序执行：

1. 基础库初始化脚本
2. `docs/数据库脚本/zuoye_submit.sql`
3. `docs/数据库脚本/核心业务一期.sql`
4. `docs/数据库脚本/核心业务三期.sql`

`zuoye_submit.sql` 主要对应：

- `ZuoyeSubmitController`
- `ZuoyeSubmitEntity`
- `ZuoyeSubmitDao.xml`

`核心业务一期.sql` 主要对应：

- `KechengController`
- `CourseChapterController`
- `CourseResourceController`
- `CourseEnrollController`
- `StudyProgressController`
- `CourseCreditRecordController`
- `CourseSettlementServiceImpl`

`核心业务三期.sql` 主要对应：

- `ExamController`
- `ExamQuestionController`
- `ExamRecordController`
- `ExamManageView.vue`
- `ExamQuestionManageView.vue`
- `ExamRecordManageView.vue`
- `ExamDetailView.vue`
