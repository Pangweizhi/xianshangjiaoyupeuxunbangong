# 核心业务实施进度

## 1. 本次落地范围

本次没有一次性完成《核心业务需求具体实现方案》全部内容，而是先完成一期主链路，让系统具备：

- 课程审核
- 章节管理
- 资源管理
- 学生选课
- 学习进度记录
- 作业关联课程/章节
- 结课发分

## 2. 后端已完成内容

### 2.1 课程模块扩展

已扩展 `kecheng` 相关模型与查询字段，新增：

- `courseStatus`
- `creditScore`
- `reviewRemark`
- `reviewTime`
- `reviewAdminId`

已补充课程接口行为：

- 教师新建/修改课程默认进入 `pending_review`
- 前台课程列表默认只展示 `approved`
- 新增 `POST /kecheng/submitReview`
- 新增 `POST /kecheng/review`

### 2.2 新增核心业务模块

已新增以下模块，并补齐 `Entity/Dao/Service/ServiceImpl/Controller/Mapper XML`：

- `course_chapter`
- `course_resource`
- `course_enroll`
- `study_progress`
- `course_credit_record`

对应控制器：

- `CourseChapterController`
- `CourseResourceController`
- `CourseEnrollController`
- `StudyProgressController`
- `CourseCreditRecordController`

### 2.3 作业模块扩展

已扩展 `zuoye`：

- `kechengId`
- `chapterId`
- `deadlineTime`
- `scoreTotal`
- `publishStatus`

当前行为：

- 前台作业列表默认只展示 `published`
- 新建作业默认 `publishStatus=published`
- 默认总分 `scoreTotal=100`
- 学生提交作业时校验截止时间

### 2.4 结课与发分逻辑

已新增 `CourseSettlementServiceImpl`，当前结课规则为：

- 课程资源完成率达到 90% 及以上
- 该课程下已发布作业全部达到 60 分及以上

满足条件后：

- 选课状态更新为 `已结课`
- 自动写入或更新 `course_credit_record`
