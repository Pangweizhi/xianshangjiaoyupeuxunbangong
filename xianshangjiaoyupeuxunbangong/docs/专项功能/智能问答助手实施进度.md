# 智能问答助手实施进度

## 1. 当前目标

本专项当前按一期“最小可用版本”推进，目标是先在现有系统内落地：

- 问AI统一入口
- 多轮会话
- 历史会话
- MySQL 持久化
- 用户隔离
- 猜你想问
- 系统导航问答
- 基于系统业务数据的基础规则问答

首期暂不强依赖完整 RAG，也不要求必须接通外部大模型后才能使用。

## 2. 一期范围拆解

### 2.1 后端

需要新增：

- `ai_chat_session` 会话表

## 5. 当前已知约束

1. 项目当前 `pom.xml` 中尚未接入 Spring AI 或阿里百炼 SDK。
2. 因此一期代码先采用“本地规则应答 + 后续可插拔模型接入”的方式落地。
3. 当前二期已接入百炼 OpenAI 兼容接口，配置通过环境变量读取：
- `QA_AI_ENABLED`
- `QA_AI_API_KEY`
- `QA_AI_BASE_URL`
- `QA_AI_MODEL`
4. 由于现有项目为 `Spring Boot 2.2 + Java 8`，本轮没有直接引入 Spring AI，而是采用百炼兼容接口直连方案。

## 6. 一期完成标准

满足以下条件即可认定一期完成：

1. 学生端和教师后台都能进入问AI页面。
2. 每个用户只能看到自己的会话和消息。
3. 可以发送消息并收到系统内 AI 回复。
4. 可以显示 2 条“猜你想问”。
5. 围绕课程、作业、考试、成绩、学习路径的问题能给出可用回答。
6. 系统无关问题会被友好拒答。
7. 后端编译通过，前后台 `vue-tsc` 检查通过。
