# 线上教育培训办公系统核心业务需求具体实现方案

## 1. 文档说明

本文依据以下两部分内容整理：

- `docs/专项功能/线上教育培训办公系统——核心业务需求文档.docx`
- 当前仓库中的后端、数据库脚本与前端实现现状

本文目标不是重复需求，而是给出“如何在现有系统上落地”的具体实现方案，并同步记录当前实际落地进度。

截至 `2026-05-03`，核心业务一期、二期、三期、四期主链路已经全部落地，当前文档更偏向“现状说明 + 剩余缺口说明”。

## 2. 现有系统实现基础

当前系统已具备一部分教学业务基础，可直接复用：

- 课程管理：`kecheng` 表，后端 `KechengController`
- 教学资料管理：`jiaoxueshipin` 表，后端 `JiaoxueshipinController`
- 作业管理：`zuoye` 表，后端 `ZuoyeController`
- 作业提交与批改：`zuoye_submit` 表，后端 `ZuoyeSubmitController`
- 公告管理：`news` 表
- 会议通知：`kaihuitongzhi` 表
- 教师、学生、管理员三类角色与 Token 鉴权
- 新前端已完成课程、资料、作业、公告、会议等基础页面

## 3. 历史缺口与当前状态

结合需求文档，原始缺口主要包括：

1. 课程审核机制
2. 课程章节模型
3. 学生选课/我的课程关系
4. 学习进度跟踪
5. 考试模块
6. 成绩汇总与课程结课判定
7. 学分发放与防重复控制
8. 围绕上述业务的后台菜单、教师端菜单和学生端页面

当前状态：

- `1` 到 `7` 已完成核心落地
- `8` 已完成主页面与主菜单落地
- 剩余未做的是更细的运营级能力，而不是主链路缺失

## 4. 总体实现思路

系统按“教-学-练-考-评-证”主线扩展，业务链路如下：

1. 教师创建课程
2. 管理员审核课程
3. 教师维护课程章节与章节资源
4. 管理员审核章节/资源
5. 学生选课进入我的课程
6. 学生按章节学习视频与课件
7. 系统记录学习进度
8. 教师发布作业与考试
9. 学生提交作业并参加考试
10. 系统汇总作业成绩、考试成绩、学习完成度
11. 满足结课规则后发放学分

## 5. 模块级实现现状

## 5.1 课程管理与审核

已落地内容：

- `kecheng` 已扩展 `course_status`、`credit_score`、审核相关字段
- 教师侧支持课程提交审核
- 管理端支持课程审核
- 学生端只展示通过审核课程

当前结论：一期已完成。

## 5.2 课程章节管理

已落地内容：

- 新增 `course_chapter`
- 后端 `CourseChapterController/Service/Dao/Entity`
- 管理端章节管理页面
- 学生端课程详情可查看章节目录

当前结论：一期已完成。

## 5.3 教学资源管理

已落地内容：

- 新增 `course_resource`
- 资源关联课程与章节
- 支持资源审核与学生侧查看
- 学习进度计算基于章节资源

当前结论：二期已完成。

## 5.4 学生选课与我的课程

已落地内容：

- 新增 `course_enroll`
- 学生支持选课
- 学生端 `MyCoursesView.vue`
- 管理端选课管理页面

当前结论：二期已完成。

## 5.5 学习进度管理

已落地内容：

- 新增 `study_progress`
- 学生学习时写入/更新进度
- 管理端支持查看学习进度
- 结课判定基于资源完成率

当前结论：二期已完成。

## 5.6 作业模块优化

已落地内容：

- `zuoye` 已关联课程/章节
- 增加截止时间、总分、发布状态
- `zuoye_submit` 已支持评分、状态、批改时间、评语
- 作业评分变更后会触发结课判定

当前结论：三期已完成。

## 5.7 考试模块

已落地内容：

- 新增 `exam`
- 新增 `exam_question`
- 新增 `exam_record`
- 学生在线考试、客观题自动判分、主观题待阅卷
- 教师批改、重置未提交作答、作废考试记录
- 支持 `allow_retake`、`max_attempt_count`、`allow_resume`

当前结论：三期已完成。

说明：

- 需求文档中建议的 `exam_paper_question`、`exam_answer` 当前未单独拆表
- 现实现用 `question_snapshot`、`answer_snapshot` 存储试卷快照与答题明细
- 这不影响主业务闭环，但属于实现方案与建议方案的差异

## 5.8 学分发放与结课判定

已落地内容：

- 新增 `course_credit_record`
- `CourseSettlementServiceImpl` 统一处理结课判定
- 规则包含：学习进度、作业成绩、考试成绩
- 满足条件后自动更新选课状态并发放学分
- 同一课程同一学生学分记录做更新而非重复插入

当前结论：四期核心结算已完成。

## 5.9 第四期成绩汇总与统计报表

本轮新增并已落地：

- `GET /courseReport/overview`
- `GET /courseReport/mySummary`
- 管理端 `DashboardView.vue` 接入第四期总览
- 学生端 `MyScoresView.vue` 改为课程维度成绩汇总

当前可查看：

- 课程平均学习进度
- 作业平均分与通过数
- 考试平均分与通过数
- 结课人数
- 学分发放数量与总学分
- 学生个人课程维度成绩汇总

当前结论：四期主链路已完成，但“运营级报表”仍未完成。

## 8. 数据迁移与兼容方案

当前仍沿用兼容式升级方案：

1. 保留现有 `kecheng`、`jiaoxueshipin`、`zuoye`、`zuoye_submit`
2. 通过新增字段和新增表完成扩展
3. 原 `jiaoxueshipin` 数据可逐步迁移到 `course_resource`
4. 前端可并存旧资料与章节资源

当前状态：兼容方案已具备，但“旧资料到新资源”的正式迁移脚本尚未提供。

## 9. 当前剩余未做内容

从核心业务需求角度看，当前剩余未做的不是主闭环，而是以下增强项：

1. 报表导出与时间区间统计
2. 更细粒度的课程结课规则配置化
3. 教师端更细粒度的考试规则批量配置
4. 考试异常中断日志
5. `exam_paper_question`、`exam_answer` 这类更规范的拆表实现
6. 旧 `jiaoxueshipin` 到 `course_resource` 的正式迁移脚本
7. 浏览器真实联调回归与数据库真实验收

## 10. 结论

当前系统已经从“课程 + 资料 + 作业”的基础教学系统，升级为完整的：

- 课程审核体系
- 章节化学习体系
- 学生选课与进度跟踪体系
- 作业与考试评价体系
- 结课与学分发放体系
- 第四期课程成绩汇总与统计总览

因此，如果按“核心业务需求四期主链路是否完成”来判断，答案是：已经完成。  
如果按“是否达到更细的运营级产品形态”来判断，答案是：还差报表导出、规则配置化、异常日志和数据迁移等增强项。
