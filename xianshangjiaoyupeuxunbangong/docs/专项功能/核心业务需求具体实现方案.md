# 核心业务需求具体实现方案

## 1. 文档说明

本文件用于说明“线上教育培训办公系统”核心业务需求在当前仓库中的具体落地方式，并同步记录当前现状。

截至 `2026-05-07`，核心业务一期、二期、三期、四期主链路已经落地，本文重点放在：

- 当前系统是如何实现的
- 还剩哪些增强项没有做
- 最近补充修复了哪些以前未登记的内容

## 2. 现有系统基础

当前系统已经具备以下基础能力：

- 课程管理
- 教学资源管理
- 作业管理
- 公告管理
- 教师、学生、管理员三类角色权限
- 新前端课程、作业、公告等基础页面

在此基础上，已经继续扩展出完整的课程学习、考试、结课、学分链路。

## 3. 总体实现思路

系统按“课程创建 -> 审核 -> 章节资源 -> 学生选课 -> 学习进度 -> 作业考试 -> 结课学分”的主链路扩展。

核心业务流程如下：

1. 教师创建课程
2. 管理员审核课程
3. 教师维护课程章节与章节资源
4. 管理员审核章节资源
5. 学生选课并进入“我的课程”
6. 学生按章节学习视频和课件
7. 系统记录学习进度
8. 教师发布作业与考试
9. 学生提交作业并参加考试
10. 系统汇总学习进度、作业成绩、考试成绩
11. 满足规则后结课并发放学分

## 5. 数据迁移与兼容方案

当前仍采用兼容式升级方案：

1. 保留原有 `kecheng`、`jiaoxueshipin`、`zuoye`、`zuoye_submit`
2. 通过新增字段和新增表完成扩展
3. 旧 `jiaoxueshipin` 数据可逐步迁移到 `course_resource`
4. 前端兼容旧资料与新章节资源并存

当前状态：

- 兼容方案已经具备
- 旧资源到新资源的正式迁移脚本仍未单独整理

## 6. 最近补充的数据脚本现状
