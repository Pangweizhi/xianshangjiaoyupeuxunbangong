# 前端重写工作区说明

## 1. 文档目的

本文用于说明新前端工作区的目录结构、职责划分、接口复用方式和当前实际落地范围。

截至 `2026-05-05`，该工作区已经承载学生端、管理端和共享层的主要新代码，不再只是一个原型目录。

## 2. 工作区位置

新前端位于仓库根目录下的 `new-frontend`：

```text
xianshangjiaoyupeuxunbangong
├─ new-frontend
│  ├─ admin-web
│  ├─ front-web
│  ├─ shared
│  ├─ package.json
│  └─ pnpm-workspace.yaml
├─ docs
├─ src
└─ ...
```

采用独立工作区的原因：

1. 不直接破坏旧前端资源，便于并行保留
2. 学生端、管理端、共享层可以拆开维护
3. 更适合持续重构和后续交接

## 3. 三个子工程的职责

### 3.1 `front-web`

面向学生端，当前已经覆盖：

- 首页
- 登录
- 课程列表 / 课程详情
- 章节视频学习页
- 我的课程
- 作业列表 / 作业详情
- 考试列表 / 考试详情
- 我的成绩
- 问 AI 页面

### 3.2 `admin-web`

面向后台管理员 / 教师端，当前已经覆盖：

- 登录
- 仪表盘
- 课程管理
- 章节管理
- 资源管理
- 选课管理
- 学习进度管理
- 作业管理
- 作业提交记录管理
- 考试管理
- 题库管理
- 阅卷管理
- 公告管理
- 公告类型管理
- 问 AI 页面

### 3.3 `shared`

用于承载前后两个新前端共享的基础能力，主要包括：

- HTTP 请求封装
- 登录态处理
- Token 持久化
- 通用类型定义
- 分页返回结构
- AI、课程、考试等共享类型

## 4. 当前工程脚本

`new-frontend/package.json` 当前主要脚本如下：

```json
{
  "scripts": {
    "dev:front": "pnpm --filter front-web dev",
    "dev:admin": "pnpm --filter admin-web dev",
    "build": "pnpm -r build"
  }
}
```

当前本地开发端口：

- `front-web`: `5173`
- `admin-web`: `5174`

## 5. 当前后端复用方式

新前端仍直接复用现有 Spring Boot 后端，不是独立重写后端。

默认后端地址：

```text
http://localhost:8080/xianshangjiaoyupeuxunbangong
```

当前主要登录接口：

- 管理员：`POST /users/login`
- 教师：`POST /jiaoshi/login`
- 学生：`POST /yonghu/login`

当前常用业务接口范围包括：

- `/dictionary/*`
- `/kecheng/*`
- `/news/*`
- `/zuoye/*`
- `/zuoyeSubmit/*`
- `/courseChapter/*`
- `/courseResource/*`
- `/courseEnroll/*`
- `/studyProgress/*`
- `/exam/*`
- `/examQuestion/*`
- `/examRecord/*`
- `/aiChat/*`

### 6.1 学生端

最近已经明确落地：

- 章节视频页布局调整
- 学习进度 `99%` / `0%` 回显问题修复
- 成绩汇总页按课程维度展示

### 6.2 管理端

最近已经明确落地：

- 资源表单必填校验
- 题库分页
- 题库按课程分类
- 考试按课程从题库选题
- 公告类型管理菜单接入
- 公告编辑保存修复

## 7. 当前仍需注意的边界

- 旧前端仍在仓库中，不要误删
- 新前端默认建立在后端接口不大改的前提下
- 页面权限展示可以前端控制，但真正权限仍以后端为准
- 数据结构变更后，需要同步更新 `shared` 类型和接口调用层
