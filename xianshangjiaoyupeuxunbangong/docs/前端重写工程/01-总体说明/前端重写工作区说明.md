# 前端重写工作区说明

## 1. 文档目的

这份文档用于说明“新前端重写工程”当前的目录结构、职责划分、接口约定和运行方式，便于后续继续开发、交接和排查问题。

## 2. 工作区位置

新前端代码不直接混在旧前端目录里，而是单独放在仓库根目录下的 `new-frontend`：

```text
xianshangjiaoyupeuxunbangong
├─ new-frontend
│  ├─ admin-web
│  ├─ front-web
│  ├─ shared
│  ├─ package.json
│  └─ pnpm-workspace.yaml
├─ docs
├─ database
└─ src
```

这样做的目的有两个：

1. 不直接破坏旧前端，方便对照迁移。
2. 新前台、新后台、共享代码可以分开维护，结构更清楚。

## 3. 三个子工程分别做什么

### 3.1 `front-web`

面向学生/前台用户的新站点，当前已经覆盖：

- 首页
- 登录
- 课程列表
- 公告列表
- 作业列表与作业详情
- 论坛列表、帖子详情、回帖
- 备课资料列表、详情、下载
- 会议通知列表、详情
- 个人中心

### 3.2 `admin-web`

面向后台运营和教师使用的新后台，当前已经覆盖：

- 管理员登录
- 教师登录
- 管理员仪表盘
- 教师工作台
- 课程管理
- 公告管理
- 学生管理
- 教师管理
- 字典管理
- 系统配置
- 作业管理
- 论坛管理
- 备课管理
- 会议管理
- 作业提交记录管理

### 3.3 `shared`

前后台共用的基础能力，主要包括：

- 登录态持久化
- Token 处理
- HTTP 请求封装
- 类型定义
- 兼容旧后端 `session` 的 `withCredentials`

## 4. 为什么要拆成共享层

旧系统前后台接口风格基本一致，重复写请求层会造成两类问题：

1. 同一个接口规则要维护两遍。
2. 登录态、上传、错误处理很容易出现前后台行为不一致。

因此把鉴权、请求、类型抽到 `shared`，前后台只关心自己的页面逻辑。

## 5. 当前后端接口约定

新前端仍然复用现有 Spring Boot 后端，没有重写后端业务主链路。

当前默认后端地址：

```text
http://localhost:8080/xianshangjiaoyupeuxunbangong
```

当前主要登录接口：

- 管理员：`POST /users/login`
- 教师：`POST /jiaoshi/login`
- 学生：`POST /yonghu/login`

当前主要通用接口：

- 文件上传：`POST /file/upload`
- 文件下载：`GET /file/download`
- 字典：`/dictionary/*`
- 课程：`/kecheng/*`
- 公告：`/news/*`
- 作业：`/zuoye/*`
- 作业提交：`/zuoyeSubmit/*`

## 6. 当前新增的后端补充

为了让“作业提交”不是只停留在前端页面上，已经补充了新的后端表和接口：

- 新表：`zuoye_submit`
- SQL 文件：
  - `docs/数据库脚本/zuoye_submit.sql`
- 主要字段：
  - 提交状态
  - 提交附件
  - 评分
  - 评语
  - 批改时间

## 7. 当前已经落地的关键能力

### 7.1 前台

- 作业详情页可上传并提交作业
- 论坛支持发帖和回帖
- 备课资料支持下载
- 会议通知支持详情查看
- 作业详情页可看到最近一次提交状态、评分、评语

### 7.2 后台

- 后台各管理页已统一到同一套基础交互
- 支持搜索、分页、重置
- 支持新增、编辑、删除
- 支持上传预览和错误提示
- 提交记录页支持批改、退回重交、批量批改、批量删除

### 7.3 教师侧

- 教师可以单独登录新版后台
- 教师菜单已按角色过滤
- 教师工作台可查看批改工作量和提交报表

## 9. 当前限制

当前这台机器上无法直接完成新前端启动验证，原因是：

- `pnpm` 不存在
- `node` 和 `npm` 执行返回“拒绝访问”

因此现阶段完成的是源码落地、结构整理和静态检查，不是本机跑通验证。
