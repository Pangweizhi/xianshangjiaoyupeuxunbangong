# 修改边界说明

为避免后续改动再次落到旧页面，统一约定如下：

- 旧前端目录 `src/main/resources/front/front` 保持只读，不再做页面或样式改动。
- 旧后台目录 `src/main/resources/admin/admin` 保持只读，不再做页面或样式改动。
- 所有“前台”相关的视觉、交互、布局改动，只允许放到 `new-frontend/front-web`。
- 所有“后台”相关的视觉、交互、布局改动，只允许放到 `new-frontend/admin-web`。
- 如果需要同步接口、类型或公共工具，只改 `shared` 或后端接口层，不回写旧前后端页面。

这条规则优先级最高。今后如果需求只说“前端”，默认指 `new-frontend` 下的新工程，不再改动旧前后端。
