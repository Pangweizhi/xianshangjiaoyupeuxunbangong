# 前端重写方案

## 1. 目标

当前项目后端接口和数据库已经可用，旧前端继续保留，新方案目标是：

- 建立独立的新学生端
- 建立独立的新管理端
- 复用现有 Spring Boot 后端
- 在不破坏旧系统运行的前提下持续替换前端体验

## 2. 当前现状判断

### 2.1 旧前端

旧学生端位于原有静态资源目录，技术风格混杂，适合“保留运行”，不适合继续大规模演进。

### 2.2 旧后台

旧后台是较老的 Vue2/Element 风格后台，功能虽全，但视觉、交互和维护体验都偏旧。

### 2.3 新前端现状

截至 `2026-05-05`，新前端已经不仅是方案，而是已经实际落地：

- `front-web`
- `admin-web`
- `shared`

并承载了核心业务页面。

## 3. 总体方案

采用“双前端 + 共享层 + 复用旧后端”的方式：

```text
new-frontend/
  front-web/
  admin-web/
  shared/
```

说明：

- `front-web` 负责学生端体验
- `admin-web` 负责后台管理和教师业务
- `shared` 负责共享请求、鉴权、类型与通用逻辑

## 4. 技术选型

当前已实际采用：

### 4.1 学生端

- `Vue 3`
- `Vite`
- `TypeScript`
- `Pinia`
- `Vue Router`
- `Axios`

### 4.2 管理端

- `Vue 3`
- `Vite`
- `TypeScript`
- `Element Plus`
- `Pinia`
- `Vue Router`
- `Axios`

## 5. 接口复用方式

新前端直接复用现有后端接口，重点做三件事：

1. 统一登录态处理
2. 统一 HTTP 封装
3. 统一共享类型维护

当前主要接口范围包括：

- 认证
- 字典
- 课程
- 公告
- 作业
- 考试
- 学习进度
- AI 问答

## 6. 页面迁移优先级

当前已经落地的迁移优先级基本是：

1. 基础入口页
2. 学生端学习链路页
3. 后台管理主链路页
4. 统计与成绩页
5. AI 问答页
6. 体验修复和文档补齐

## 8. 当前方案阶段判断

前端重写目前已经不再处于“方案讨论阶段”，而是处于“继续维护、继续补齐、继续回归验证”的阶段。

也就是说：

- 基础架构已定
- 主要页面已定
- 后续重点是增量迭代和问题收口

## 9. 后续建议

后续如果继续推进，优先建议做：

1. 浏览器端完整回归
2. 页面交互统一细节
3. 旧页面与新页面切换策略梳理
4. 剩余存量文档同步
