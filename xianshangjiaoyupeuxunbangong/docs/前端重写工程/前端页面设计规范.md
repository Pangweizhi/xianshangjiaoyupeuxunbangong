# 前端页面设计规范

## 1. 目标

本文用于统一学生端和管理端页面的视觉与结构标准，避免同一套系统出现多套风格、多套信息层级和多套交互习惯。

## 2. 总体设计原则

统一遵守：

- 风格统一
- 层次清晰
- 结构稳定
- 信息优先
- 交互明确
- 兼顾扩展

## 3. 学生端页面规范

### 3.1 适用范围

学生端页面面向学习与内容浏览，重点是：

- 内容展示
- 学习操作
- 个人进度
- 成绩结果

### 3.2 页面结构建议

学生端常见页面结构包括：

- 首页
- 登录 / 注册
- 列表页
- 详情页
- 学习页
- 个人中心页

### 3.3 风格要求

- 保持内容门户和学习平台气质
- 信息区块清晰分层
- 适当留白
- 不做过重的后台式表格堆叠
- 重点区域使用明确视觉焦点

### 3.4 学习页要求

对于课程视频、考试、作业等学习页，要求：

- 主任务区域必须突出
- 辅助信息区不能抢主内容
- 左右分栏时，主内容区占比明显更大
- 关键状态如进度、倒计时、提交状态需要稳定可见

当前项目中，典型对应包括：

- 章节视频播放页
- 在线考试页
- 作业详情页

## 4. 管理端页面规范

### 4.1 适用范围

管理端页面面向管理员和教师，重点是：

- 效率
- 清晰
- 稳定
- 一致

### 4.2 页面结构建议

管理端常见页面结构包括：

- 登录页
- 仪表盘
- 列表管理页
- 表单页 / 弹窗
- 详情页
- 系统工具页

### 4.3 风格要求

- 与学生端保持同一主题方向
- 但信息密度允许更高
- 操作入口要明确
- 表格、筛选、表单样式统一
- 不允许页面之间风格割裂

### 4.4 后台布局要求

当前项目后台特别要求：

- 左侧菜单与右侧内容区域颜色过渡自然
- 避免明显生硬分割线
- 主色与交互色与学生端保持一致方向
- 按钮颜色要保证可读性，不能出现文字看不清的问题

## 5. 通用页面模板

### 5.1 列表页模板

建议包含：

- 页面标题
- 简短说明
- 搜索筛选区
- 操作区
- 列表区
- 分页区

### 5.2 表单页模板

建议包含：

- 标题
- 说明
- 分组表单项
- 必填标识
- 校验提示
- 提交按钮

### 5.3 详情页模板

建议包含：

- 标题区
- 元信息区
- 主内容区
- 附件或扩展信息区
- 返回入口

### 5.4 仪表盘模板

建议包含：

- 统计卡片
- 关键图表
- 待办区
- 快捷入口

## 6. 视觉细节要求

- 标题层级清晰
- 文本对比足够
- 间距统一
- 圆角统一
- 阴影克制
- 主色只用于重点强调
- 状态色只表达状态，不滥用

## 7. 响应式要求

- 桌面端优先
- 移动端可用
- 左右分栏在小屏下能自动改为单列或上下结构
- 表格页在窄屏下至少能保证核心信息不失真

## 8. 当前项目特别补充

截至 `2026-05-05`，结合当前系统实际情况，额外强调：

- 视频页、考试页、作业页都属于“主任务页”，布局必须服务主任务
- 题库管理、考试管理、资源管理、公告管理属于“后台核心管理页”，需要统一交互骨架
- 学生端和后台端虽然场景不同，但主题气质不能完全断裂

## 9. 验收标准

页面设计规范落地后，至少应满足：

- 学生端与管理端风格统一但各有侧重
- 新页面可以复用现有模板快速搭建
- 关键业务页面信息层级稳定
- 后续扩展时不需要每次重新定义页面结构
