# 前端工程规范

## 1. 目标

本文用于统一新前端工程中的目录结构、接口组织、组件拆分、样式变量、命名方式和基本开发约束。

适用范围：

- `new-frontend/front-web`
- `new-frontend/admin-web`
- `new-frontend/shared`

## 2. 总体原则

开发时统一遵守以下原则：

1. 复用现有后端接口，不随意新增重复接口层
2. 学生端、管理端各自负责页面，通用能力放到 `shared`
3. 页面逻辑、请求逻辑、类型定义分层清晰
4. 优先维护可读性和可扩展性，不追求过度抽象
5. 新增功能时同步更新相关文档

## 3. 工程拆分规范

### 3.1 子工程职责

- `front-web`：学生端页面、学生端路由、学生端业务 API
- `admin-web`：管理端 / 教师端页面、后台路由、后台业务 API
- `shared`：共享 HTTP 封装、共享类型、共享工具

### 3.2 不允许的做法

- 不把学生端页面代码直接写进 `admin-web`
- 不把管理端页面代码直接写进 `front-web`
- 不在两个子工程中复制同一份基础工具代码而不沉到 `shared`

## 4. 推荐目录结构

每个前端子工程建议遵循：

```text
src/
├─ api/
├─ components/
├─ layouts/
├─ router/
├─ stores/
├─ styles/
├─ views/
└─ main.ts
```

`shared` 建议保持：

```text
src/
├─ http.ts
├─ storage.ts
├─ types.ts
└─ index.ts
```

## 5. 组件规范

### 5.1 组件分层

- 基础组件：纯展示或通用交互
- 业务组件：带业务语义的复用块
- 页面组件：路由直接挂载的完整页面

### 5.2 组件原则

- 单一职责
- 命名清晰
- 尽量减少隐式依赖
- 默认值合理
- 能复用时再抽，不为抽象而抽象

### 5.3 推荐通用组件方向

- `PageHeader`
- `SearchBar`
- `StatCard`
- `DataTable`
- `FormDialog`
- `UploadField`
- `EmptyState`

## 6. API 封装规范

### 6.1 请求层要求

- 所有请求统一走共享 HTTP 封装
- 统一 `baseURL`
- 统一超时
- 统一 Token 注入
- 统一错误处理

### 6.2 API 模块划分建议

- `auth`
- `course`
- `homework`
- `exam`
- `notice`
- `dictionary`
- `ai`

### 6.3 命名建议

- `getXxxPage`
- `getXxxDetail`
- `createXxx`
- `updateXxx`
- `deleteXxx`
- `submitXxx`

## 7. 类型定义规范

- 共享业务类型优先放到 `shared/src/types.ts`
- 页面内部临时类型可以先局部声明
- 一旦被多个页面复用，必须提升到共享层
- 类型命名尽量贴近后端实体语义

## 8. 路由规范

- 页面必须通过路由统一接入，不要写散乱入口
- 学生端与管理端路由分开维护
- 需要登录的页面明确加鉴权元信息
- 菜单路径、路由名称、页面标题尽量保持一致

## 9. 状态管理规范

- 全局登录态、用户态放 `store`
- 页面局部状态尽量留在组件内部
- 不为了一次性页面强行建全局状态

## 10. 样式规范

### 10.1 样式原则

- 统一颜色变量
- 统一圆角
- 统一间距
- 统一标题层级
- 统一按钮和表单交互

### 10.2 变量建议

- `--color-primary`
- `--color-success`
- `--color-warning`
- `--color-danger`
- `--color-info`
- `--color-bg`
- `--color-surface`
- `--color-text-primary`
- `--color-text-secondary`
- `--radius-md`

### 10.3 当前项目的实际要求

- 管理端整体风格要与学生端保持同一主题方向
- 避免学生端和后台端完全割裂成两套审美
- 后台允许更偏功能化，但不能显得陈旧或杂乱

## 11. 页面开发规范

### 11.1 列表页

建议统一包含：

- 页面标题
- 搜索区
- 操作区
- 表格区
- 分页区

### 11.2 表单页 / 弹窗

建议统一包含：

- 标题
- 必填标识
- 校验提示
- 提交 / 取消操作

### 11.3 详情页

建议统一包含：

- 标题区
- 基础信息区
- 主体内容区
- 附件或扩展信息区

## 12. 提交变更时的同步要求

涉及以下变更时，必须同步更新文档：

- 新增核心页面
- 新增共享类型
- 新增数据库脚本依赖
- 改动重要业务流程
- 修改导航结构
