# 前端工程规范

## 1. 目标

这份文档把前端重写工程里真正影响开发效率和扩展性的基础规则统一起来，包括组件、接口、配色、字体、路由、权限、交互、初始化和迁移。

## 2. 目录与工程初始化

### 2.1 推荐工程拆分

- `front-web`
- `admin-web`

### 2.2 初始化必须具备

- 路由
- 状态管理
- 请求层
- 布局
- 主题变量
- 公共组件
- 类型定义

### 2.3 推荐目录

```text
src/
├─ api/
├─ components/
├─ layouts/
├─ router/
├─ stores/
├─ styles/
├─ types/
├─ utils/
└─ views/
```

## 3. 组件规范

### 3.1 组件分层

- 基础组件
- 通用业务组件
- 页面级组件

### 3.2 组件原则

- 单一职责
- 高内聚低耦合
- 可配置
- 默认值合理

### 3.3 建议公共组件

- `AppLayout`
- `PageHeader`
- `SearchBar`
- `StatCard`
- `DataTable`
- `FormDialog`
- `EmptyState`
- `UploadField`
- `RichTextEditor`
- `ConfirmButton`

## 4. 接口封装规范

### 4.1 请求层要求

- 单一请求入口
- 统一 `baseURL`
- 统一超时
- 统一请求头
- 统一错误处理
- 统一 token 注入

### 4.2 接口拆分建议

- `auth`
- `user`
- `teacher`
- `student`
- `course`
- `homework`
- `forum`
- `notice`
- `dictionary`
- `config`
- `file`

### 4.3 命名规范

- `login`
- `logout`
- `register`
- `getXxxPage`
- `getXxxDetail`
- `createXxx`
- `updateXxx`
- `deleteXxx`

## 6. 路由与权限规范

### 6.1 路由分区

前台：

- `/`
- `/login`
- `/register`
- `/course`
- `/news`
- `/homework`
- `/forum`
- `/notice`
- `/teacher`
- `/student`

后台：

- `/dashboard`
- `/users`
- `/teachers`
- `/students`
- `/courses`
- `/homeworks`
- `/lessons`
- `/forums`
- `/notices`
- `/dictionary`
- `/config`
- `/backup`

### 6.2 权限模型

- 角色权限
- 页面权限
- 菜单权限

### 6.3 路由守卫

- 未登录跳登录页
- 已登录不能回登录页
- token 失效自动退出
- 不同角色进入不同默认首页

## 7. 交互与动效规范

### 7.1 交互原则

- 有反馈
- 有状态
- 有结果提示
- 克制

### 7.2 常见交互

- hover
- active
- focus
- loading
- empty
- success
- warning
- error

### 7.3 动效原则

- 轻量
- 短时
- 服务于信息
- 不喧宾夺主

### 7.4 推荐动效

- 页面淡入
- 卡片轻微上浮
- 弹窗平滑展开
- 列表项渐入

## 9. 前端迁移实施

### 9.1 阶段划分

1. 基础工程
2. 视觉骨架
3. 核心业务
4. 后台模块
5. 优化与替换

### 9.2 迁移原则

- 后端不动
- 旧前端保留
- 新前端并行
- 先核心后扩展

## 10. 交付标准

工程规范完成后，至少应满足：

- 工程能快速初始化
- 组件能稳定复用
- 接口层清晰
- 路由权限清楚
- 样式变量统一
- 后续页面开发不会失控
