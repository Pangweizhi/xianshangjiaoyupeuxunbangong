# 新前端启动说明

## 1. 适用范围

本文只针对当前项目：

```text
xianshangjiaoyupeuxunbangong
```

不要混用其他项目的前端启动方式。

## 2. 启动前准备

本机需要具备：

1. `Node.js`
2. `pnpm`
3. 可正常运行的当前 Spring Boot 后端

可先检查版本：

```bash
node -v
pnpm -v
```

如果没有安装 `pnpm`，可以执行：

```bash
npm install -g pnpm
```

## 3. 新前端目录

新前端目录位于：

```bash
cd new-frontend
```

该目录是一个 `pnpm workspace`，包含：

- `front-web`
- `admin-web`
- `shared`

## 4. 最常用启动方式

先启动后端，再进入新前端目录执行：

```bash
cd new-frontend
pnpm install
pnpm dev:front
pnpm dev:admin
```

访问地址：

- 学生端：`http://localhost:5173`
- 管理端：`http://localhost:5174`
- 后端：`http://localhost:8080/xianshangjiaoyupeuxunbangong`

说明：

- `pnpm install` 只需要在首次安装依赖或依赖变化后执行
- `pnpm dev:front` 启动学生端
- `pnpm dev:admin` 启动管理端

## 5. 单独启动某一端

只看学生端：

```bash
cd new-frontend
pnpm dev:front
```

只看管理端：

```bash
cd new-frontend
pnpm dev:admin
```

## 6. 为什么必须先启动后端

当前新前端不是独立系统，而是直接调用现有 Spring Boot 后端接口。

如果后端没有启动，常见现象包括：

- 登录失败
- 列表为空
- 上传失败
- 页面提示网络错误

## 7. 数据库脚本要注意什么

当前新前端依赖的数据库补充脚本不止一份，至少应关注：

- `docs/数据库脚本/zuoye_submit.sql`
- `docs/数据库脚本/核心业务一期.sql`
- `docs/数据库脚本/核心业务三期.sql`
- `docs/数据库脚本/题库初始化.sql`
- `docs/数据库脚本/题库课程题目补充.sql`
- `docs/数据库脚本/智能问答助手一期.sql`

如果数据库结构不完整，常见表现是：

- 页面能打开，但提交失败
- 题库为空或课程分类异常
- 学习进度无法正常记录
- AI 会话页面报错

## 8. 当前建议的验证顺序

启动成功后，建议至少验证以下页面：

1. 学生端登录
2. 课程列表 / 课程详情
3. 章节视频页
4. 我的课程
5. 作业列表
6. 考试列表
7. 管理端课程管理
8. 管理端资源管理
9. 管理端题库管理
10. 管理端公告管理 / 公告类型管理

## 9. 按当前代码状态建议重点检查的点

最近补过的功能，建议启动后优先看：

- 视频学习进度是否能正常回显为 `100%`
- 视频播放页信息区是否在右侧
- 资源新增表单是否有必填提示
- 题库是否支持分页与按课程筛选
- 公告编辑后保存是否生效

## 10. 构建说明

工作区根目录支持：

```bash
cd new-frontend
pnpm build
```

该命令会递归执行子工程构建。

如果构建失败，优先检查：

- `node` 版本
- `pnpm` 版本
- 依赖是否已正确安装
- 本机权限或安全软件是否拦截 `vite/esbuild`
