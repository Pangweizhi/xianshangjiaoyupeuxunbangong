# 新前端启动说明

这份文档只针对当前项目：

```text
xianshangjiaoyupeuxunbangong
```

不要参考“另外一个项目的新前端启动教程”。那个目录对应的是别的项目，启动方式和当前项目不一样。

## 1. 1 分钟启动

先启动 Spring Boot 后端，再进入新前端目录执行：

```bash
cd new-frontend
pnpm install
pnpm dev:front
pnpm dev:admin
```

访问地址：

- 前台：`http://localhost:5173`
- 后台：`http://localhost:5174`
- 后端：`http://localhost:8080/xianshangjiaoyupeuxunbangong`

只记 4 件事：

1. `pnpm install` 只在第一次启动或依赖变化后执行
2. `pnpm dev:front` 启动前台
3. `pnpm dev:admin` 启动后台
4. 前端依赖当前 Spring Boot 后端，后端没起时页面可能能打开，但数据会失败

页面没数据时先查：

1. 后端是否真的启动了
2. 数据库是否连接正常
3. `zuoye_submit` 相关表是否已按脚本处理

## 2. 常用启动方式

如果你只看前台，就执行：

```bash
cd new-frontend
pnpm dev:front
```

如果你只看后台，就执行：

```bash
cd new-frontend
pnpm dev:admin
```

如果前后台都要看，就开两个终端分别执行：

```bash
pnpm dev:front
pnpm dev:admin
```

## 3. 启动前要满足什么条件

本机需要有：

1. `Node.js`
2. `pnpm`
3. 可正常运行的当前 Spring Boot 后端

快速检查命令：

```bash
node -v
pnpm -v
```

如果命令都能输出版本号，说明前端运行环境基本就绪。

如果本机还没有 `pnpm`，可以先执行：

```bash
npm install -g pnpm
```

## 4. 为什么必须先起后端

当前新前端不是独立系统，它直接调用当前 Spring Boot 后端接口。

后端默认地址：

```text
http://localhost:8080/xianshangjiaoyupeuxunbangong
```

后端没启动时，常见现象是：

- 登录失败
- 列表为空
- 上传失败
- 页面提示网络错误

## 5. 数据库还要注意什么

作业提交流程依赖 `zuoye_submit` 相关表。

脚本位置：

- `docs/数据库脚本/zuoye_submit.sql`

使用规则：

1. 无论是新库还是旧库，统一执行 `zuoye_submit.sql`
2. 这份脚本已经同时支持建表和补齐旧表缺失字段

如果这部分没处理好，页面可能能打开，但提交、批改、评分这条链路会出问题。

## 6. 新前端目录在哪里

项目根目录下有：

```text
new-frontend
```

进入方式：

```bash
cd new-frontend
```

这个目录下是一个 `pnpm workspace`，包含：

- `front-web`
- `admin-web`
- `shared`

所以依赖安装和启动命令，都建议在 `new-frontend` 这一层执行。

## 7. 第一次启动为什么要先执行 pnpm install

第一次启动前，需要先安装依赖：

```bash
cd new-frontend
pnpm install
```

这一步会把 `front-web`、`admin-web`、`shared` 需要的依赖一起装好。

一般情况下：

- 第一次启动要执行
- 删除过 `node_modules` 后要重新执行
- 改过依赖配置后要重新执行

平时反复启动前端，不需要每次都执行。

## 8. 怎么启动前台

在 `new-frontend` 目录执行：

```bash
pnpm dev:front
```

访问地址：

```text
http://localhost:5173
```

适合验证：

- 首页
- 学生登录
- 课程浏览
- 公告浏览
- 作业查看与提交
- 论坛发帖回帖
- 备课资料下载
- 会议通知查看
- 个人中心

## 9. 怎么启动后台

在 `new-frontend` 目录执行：

```bash
pnpm dev:admin
```

访问地址：

```text
http://localhost:5174
```

适合验证：

- 管理员登录
- 教师登录
- 仪表盘
- 课程、公告、学生、教师、字典、配置管理
- 作业、论坛、备课、会议管理
- 提交记录管理、批改、退回重交、批量批改

## 10. 前后台都要看时怎么开

需要两个终端，都先进入 `new-frontend`。

第一个终端：

```bash
pnpm dev:front
```

第二个终端：

```bash
pnpm dev:admin
```

不要在同一个终端里先后输入两个开发命令，因为第一个命令会一直占用当前终端。

## 11. 只想确认能不能构建

在 `new-frontend` 目录执行：

```bash
pnpm build
```

这会同时构建前台和后台。

## 12. 登录账号怎么区分

按当前接口约定：

- 管理员登录：`/users/login`
- 教师登录：`/jiaoshi/login`
- 学生登录：`/yonghu/login`

因此：

1. 前台登录用学生账号
2. 后台如果选择管理员，就用管理员账号
3. 后台如果选择教师，就用教师账号

## 13. 启动后建议先验什么

建议按这个顺序快速确认：

1. 先看登录是否成功
2. 再看首页或列表是否有数据
3. 再看上传是否正常
4. 最后走一次作业提交和教师批改流程

如果这几步都通过，说明当前核心链路基本正常。

## 14. 当前前台界面状态

截至目前，新前台已经完成一轮界面优化，当前启动后应重点看到这些变化：

- 首页已经改成内容门户结构，不再是“重写说明页”
- 顶部导航、登录入口、个人中心入口已经统一到新版公共布局
- 课程、公告、作业、论坛、备课、会议页面都已统一到更偏内容平台的展示风格
- 多余的开发说明文字已经删除，页面文案应以真实业务表达为主

如果启动后看到的仍然是大量“新前端重写”“下一阶段”“保留原接口”之类说明文字，说明你看到的不是当前这版代码。

## 15. 常见问题

### 15.1 pnpm 命令不存在

优先检查：

1. 是否安装了 `pnpm`
2. 是否重新打开了终端
3. 环境变量是否生效

### 15.2 页面能打开但没有数据

优先检查：

1. 后端是否启动
2. 数据库是否连接正常
3. 登录状态是否有效
4. 接口是否报 404、500 或跨域错误

### 15.3 作业提交失败

优先检查：

1. `zuoye_submit` 表是否已准备好
2. `/file/upload` 接口是否可用
3. 当前登录是否为学生账号

### 15.4 后台看不到提交记录

优先检查：

1. 学生端是否真的提交成功
2. `/zuoyeSubmit/page` 是否有数据
3. 教师登录后的过滤条件是否把数据筛掉了
